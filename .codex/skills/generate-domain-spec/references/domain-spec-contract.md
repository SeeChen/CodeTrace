# Domain Spec Contract

Use this reference when drafting one domain's specification set.

## Typical Files

1. `SA.md`
2. `layer-core.md`
3. `layer-dao.md`
4. `layer-biz.md`
5. `layer-facade.md`

## Minimum Coverage

Each domain output should make clear:

- scope
- responsibilities
- internal layers
- dependencies
- failure paths
- open questions
