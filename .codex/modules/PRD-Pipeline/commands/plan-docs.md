# Command: plan-docs

## Purpose

Plan the project-specific documentation structure before generating detailed spec documents.

## Read First

1. `docs/PRD.md`
2. `docs/Workflow.md`
3. `.codex/modules/PRD-Pipeline/docs/PRD-to-Coding-Orchestration.md`
4. `.codex/modules/PRD-Pipeline/rules/doc-scope.md`
5. `.codex/modules/PRD-Pipeline/skills/plan-doc-structure/SKILL.md`
6. `.codex/modules/PRD-Pipeline/memory/workflow-memory.md`

## Execute

1. Assess project boundary, scale, and shape.
2. Decide which document phases and files are required.
3. Produce the first-pass target directory structure.
4. Mark documents as required, optional, deferred, or unnecessary.
5. Recommend the next command or agent.

## Output

- Planning update in `.codex/modules/PRD-Pipeline/docs/PRD-to-Coding-Orchestration.md` or `specs/summary.md`
- A project-specific document tree and document list

## Guardrails

- Do not copy the example structure without analysis.
- Do not generate detailed specs in this step.
- Keep all scoping decisions traceable to the PRD and workflow.
