# Doc Structure Contract

Use this reference when drafting a first-pass document plan.

## Required Sections

1. Purpose
2. Source Documents
3. Project Boundary Assessment
4. Document Phase Plan
5. Target Directory Structure
6. Required Documents
7. Optional or Deferred Documents
8. Entry Gates
9. Recommended Next Step
10. Open Questions

## Required Classification Labels

Use one of:

- Required
- Optional
- Deferred
- Unnecessary for Now

## Minimum Decision Criteria

For each major decision, explain at least one of:

- requirement coverage
- complexity management
- architecture boundary clarity
- testing readiness
- team reviewability
