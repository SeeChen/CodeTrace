# CodeTrace-AI

An AI-first repository that demonstrates a **reusable delivery pipeline**: a system of agents, skills, commands, rules, and memory that turns one PRD into architecture, build contracts, code, tests, and acceptance evidence.

The product used to demonstrate the pipeline is **CodeTrace**, a lightweight, local-first, zero-dependency Python function-tracing framework (see [docs/PRD.md](docs/PRD.md)). The real deliverable of this repository is the pipeline itself, under `.claude/`; `src/` and `tests/` are meant to be generated by running it.

## Active Flow

`PRD -> Intent Pack -> SA -> Build Spec -> Task Slices -> Coding -> Verify -> Accept`

| Stage | Command | Output |
| --- | --- | --- |
| 0 Init | `/pipeline-init` | `specs/intent/brief.md` |
| 1 SA | `/generate-sa` | `specs/architecture/SA.md` |
| 2 Build Spec | `/generate-spec` | `specs/build/*` |
| 3 Slice | `/slice-work` | `specs/build/tasks.md` |
| 4 Implement | `/implement` | `src/`, `tests/` |
| 5 Verify | `/verify` | verification evidence |
| 6 Accept | `/accept` | `specs/acceptance/*` |

The unified entry point is **`/seechen`** (for example `/seechen --run`, `/seechen --sa`, or a natural-language request).

## Overview

The repository emphasizes:

- **PRD-Driven**: `docs/PRD.md` is the only product-level source of truth; every downstream artifact derives from upstream ones
- **Specification-Driven**: freeze architecture and build contracts before coding, so implementation follows task slices rather than broad reinterpretation
- **Reusable AI Constraints**: agents, skills, commands, rules, and memory are version-controlled under `.claude/`
- **Resumable Workflow**: stage state lives in `.claude/memory/` so a run can continue from the last completed stage

## Getting Started

This repository is designed for AI-assisted development in Claude Code, where the document system comes first and code is generated from frozen specifications.

### Prerequisites

- Claude Code (or an AI tool that reads repository files and follows structured instructions)
- A project PRD as the primary source of truth

### Usage

1. Read the [PRD](docs/PRD.md), [Workflow](docs/Workflow.md), and [CLAUDE.md](CLAUDE.md).
2. Run `/seechen --run` to execute the full flow, or run a single stage such as `/seechen --sa`.
3. Inspect the reusable assets under `.claude/`:
   - `agents/` for stage role owners
   - `skills/` for per-stage execution guides
   - `commands/` for slash-command entry points
   - `rules/` for repository constraints
   - `memory/` for durable pipeline state
   - `docs/` for the pipeline interface and calling contract
4. Review the generated outputs under `specs/` as each stage completes.

## Current Scope

The repository currently provides the orchestration system for the full delivery flow (intent, architecture, build spec, task slices, coding, verification, acceptance). The pipeline starts from a clean baseline: generated outputs under `specs/`, plus `src/` and `tests/`, are produced by running the pipeline against `docs/PRD.md`.

## Contributing

Contributions should preserve the AI-first workflow. Please:

1. Follow the rules in `.claude/rules/`.
2. Keep reusable system-constraint files in English unless there is a documented exception.
3. Use a task-appropriate branch instead of reusing an unrelated branch.
4. Update `.claude/memory/pipeline-state.md` after each completed stage.
