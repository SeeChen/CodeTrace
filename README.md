# CodeTrace-AI

An AI-first repository that demonstrates a **reusable delivery pipeline**: a system of agents, skills, commands, rules, and memory that turns one PRD into architecture, build contracts, code, tests, and acceptance evidence — then keeps going with a **self-converging audit/hardening loop** until the result clears a measurable quality bar.

The product used to demonstrate the pipeline is **CodeTrace**, a lightweight, local-first, zero-dependency Python function-tracing framework (see [docs/PRD.md](docs/PRD.md)). The real deliverable of this repository is the pipeline itself, under `.claude/`; `src/` and `tests/` are generated by running it.

## Active Flow

`PRD -> Intent Pack -> SA -> Build Spec -> Task Slices -> Coding -> Verify -> Accept`

| Stage | Command | Output |
| --- | --- | --- |
| 0 Init | `/pipeline-init` | `specs/intent/brief.md` |
| 1 SA | `/generate-sa` | `specs/architecture/SA.md` |
| 2 Build Spec | `/generate-spec` | `specs/build/*` |
| 3 Slice | `/slice-work` | `specs/build/tasks.md` |
| 4 Implement | `/implement` | `src/`, `tests/` |
| 5 Verify | `/verify` | verification evidence |
| 6 Accept | `/accept` | `specs/acceptance/*` |

The unified entry point is **`/seechen`** (for example `/seechen --run`, `/seechen --sa`, or a natural-language request).

### Convergence Loop (post-acceptance hardening)

Once a milestone is green, **`/converge`** keeps iterating — `audit -> score -> stop-check -> re-plan -> implement -> verify` — until a composite quality bar is met, then stops. It is designed to satisfy the loop-engineering primitives:

- **Driver**: repetition is delegated to a native scheduler (`/loop` or `ScheduleWakeup`); `/converge --resume` runs exactly one idempotent round and signals whether another should follow.
- **Goal / score**: 70% machine-checkable gates (tests, coverage, mutation, lint, types, complexity) + 30% evidence-cited review; the loop cannot converge on self-assessment alone.
- **Safety**: a regression guard reverts any round that worsens a gate; stop conditions (converged / plateau / budget) guarantee termination; genuine ambiguity escalates to a human checkpoint instead of a guess.

See [.claude/docs/Convergence-Loop.md](.claude/docs/Convergence-Loop.md).

## Proven Results

The pipeline was run end to end against `docs/PRD.md` to generate the CodeTrace MVP, then hardened by the convergence loop until it converged:

| Round | Composite | Mutation | Complexity | Tests | Action |
| --- | --- | --- | --- | --- | --- |
| 1 | 0.82 | unmeasured | `_execute` D(21) | 43 | fixed a type-safety hole |
| 2 | 0.97 | 64.5% | `_execute` A(5) | 46 | refactored `_execute`; covered session paths |
| 3 | 0.97 | **77.3%** ✅ | A | 77 | +31 assertion-hardening tests |

**Final gates (all measured):** tests 77/77 · line coverage 99% · **mutation 77.3% (≥70%)** · ruff clean · mypy 0 errors · complexity no block worse than B.

The loop surfaced what a green suite hid — a type hole, a complexity hotspot, and a **coverage illusion** (99% of lines executed, but only 64.5% of logic actually asserted). It escalated when mutation testing could not run on native Windows (measured via WSL) and stopped at the defined bar rather than polishing forever.

## Overview

The repository emphasizes:

- **PRD-Driven**: `docs/PRD.md` is the only product-level source of truth; every downstream artifact derives from upstream ones
- **Specification-Driven**: freeze architecture and build contracts before coding, so implementation follows task slices rather than broad reinterpretation
- **Self-Converging**: a post-acceptance audit/hardening loop drives quality against machine-checkable gates until a bar is met, with a regression guard and guaranteed termination
- **Reusable AI Constraints**: agents, skills, commands, rules, and memory are version-controlled under `.claude/`
- **Resumable Workflow**: stage and loop state live in `.claude/memory/` so a run can continue from the last completed step

## Getting Started

This repository is designed for AI-assisted development in Claude Code, where the document system comes first and code is generated from frozen specifications.

### Prerequisites

- Claude Code (or an AI tool that reads repository files and follows structured instructions)
- A project PRD as the primary source of truth

### Usage

1. Read the [PRD](docs/PRD.md), [Workflow](docs/Workflow.md), and [CLAUDE.md](CLAUDE.md).
2. Run `/seechen --run` to execute the full flow, or run a single stage such as `/seechen --sa`.
3. Run `/converge --run` to harden a green milestone with the convergence loop.
4. Inspect the reusable assets under `.claude/`:
   - `agents/` for stage role owners
   - `skills/` for per-stage execution guides
   - `commands/` for slash-command entry points
   - `rules/` for repository constraints
   - `memory/` for durable pipeline and loop state
   - `docs/` for the pipeline interface and the convergence-loop reference
5. Review the generated outputs under `specs/` (including `specs/audit/` for loop evidence).

## Current Scope

The repository provides the orchestration system for the full delivery flow (intent, architecture, build spec, task slices, coding, verification, acceptance) plus the post-acceptance convergence loop. It has been demonstrated end to end: running the pipeline against `docs/PRD.md` generated the CodeTrace MVP under `src/`/`tests/`, and the convergence loop hardened it to a measured, converged state (see [Proven Results](#proven-results)). Re-running against a changed PRD regenerates the outputs from the frozen specifications.

## Contributing

Contributions should preserve the AI-first workflow. Please:

1. Follow the rules in `.claude/rules/`.
2. Keep reusable system-constraint files in English unless there is a documented exception.
3. Use a task-appropriate branch instead of reusing an unrelated branch.
4. Update `.claude/memory/pipeline-state.md` (and `convergence-state.md` during loop runs) after each completed step.
