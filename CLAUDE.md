# CodeTrace-AI

This repository demonstrates an **AI-first delivery pipeline**: a reusable system of agents, skills, commands, rules, and memory that turns one PRD into architecture, build contracts, code, tests, and acceptance evidence.

- The **product being demonstrated** is CodeTrace, a lightweight, local-first, zero-dependency Python function-tracing framework. Its PRD is the single product-level source of truth.
- The **real deliverable of this repo** is the pipeline itself (under `.claude/`). The `src/` and `tests/` are meant to be generated by running the pipeline, proving the workflow end to end.

## Active Flow

`PRD -> Intent Pack -> SA -> Build Spec -> Task Slices -> Coding -> Verify -> Accept`

| Stage | Command | Skill | Agent | Output |
| --- | --- | --- | --- | --- |
| 0 init | `/pipeline-init` | normalize-prd | prd-analyst-agent | `specs/intent/brief.md` |
| 1 sa | `/generate-sa` | generate-sa | architect-agent | `specs/architecture/SA.md` |
| 2 spec | `/generate-spec` | generate-build-spec | spec-builder-agent | `specs/build/*` |
| 3 slice | `/slice-work` | slice-build-tasks | spec-builder-agent | `specs/build/tasks.md` |
| 4 implement | `/implement` | implement-from-task | coding-agent | `src/`, `tests/` |
| 5 verify | `/verify` | verify-build | qa-acceptance-agent | verification evidence |
| 6 accept | `/accept` | accept-milestone | qa-acceptance-agent | `specs/acceptance/*` |

The unified entry point is **`/seechen`** (e.g. `/seechen --run`, `/seechen --sa`, or a natural-language request). `/prd-pipeline` is a deprecated alias.

## Key Locations

- `docs/PRD.md` — product source of truth; `docs/Workflow.md` — full stage definitions
- `.claude/commands/` — slash commands (stage entry points)
- `.claude/skills/` — per-stage execution guides
- `.claude/agents/` — stage role owners
- `.claude/rules/` — non-negotiable workflow and generation constraints
- `.claude/memory/` — durable state: `pipeline-state.md`, `frozen-decisions.md`, `open-questions.md`, `implementation-log.md`
- `.claude/docs/` — pipeline interface and calling contract
- `specs/` — generated pipeline outputs

## Operating Rules

1. `docs/PRD.md` is the only product-level source of truth; every downstream artifact derives from upstream ones.
2. Freeze architecture (`SA.md`) before generating build specs; freeze build specs before coding.
3. Coding follows task slices, not broad PRD reinterpretation; record deviations in `memory/implementation-log.md`.
4. Read `memory/pipeline-state.md` before starting and update it after each completed stage.
5. Keep agents and skills compact and docs-backed (see `.claude/rules/agent-skill-design.md`).
6. Preserve unresolved choices in `memory/open-questions.md` instead of guessing.
